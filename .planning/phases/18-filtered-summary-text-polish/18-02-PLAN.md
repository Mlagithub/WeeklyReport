# Phase 18 Plan 02: Filtered Summary Feature

## Objective
Add AI Summary button to filtered results page for team leaders.

## Context
- **Reuses:** `call_ai_api`, `fetch_user_records` pattern from summary_utils.py
- **Integration:** manage_records.html (find page), routes.py

## Tasks

### Task 1: Add filtered summary utility function
**Type:** auto
**File:** summary_utils.py
**Action:**
- Add `fetch_filtered_records()` function - similar to `fetch_user_records` but for multiple users
- Add `assemble_filtered_prompt()` - groups records by user
- Add `generate_filtered_summary()` - orchestrates the above
**Verification:** Functions exist with proper signatures

### Task 2: Add /filtered-summary route
**Type:** auto
**File:** routes.py
**Action:**
- Add POST /filtered-summary route
- Accept filter params (user, groups, time_range)
- Check user is team leader (has role or is_admin)
- Call `generate_filtered_summary`
- Return JSON with summary HTML
**Verification:** Route returns filtered summary

### Task 3: Add AI Summary button to manage_records.html
**Type:** auto
**Files:** templates/manage_records.html
**Action:**
- Add "AI 总结" button (visible for team leaders)
- Add modal for summary display
- Add JavaScript to call /filtered-summary with current filter params
- Display filter criteria summary in modal header
**Verification:** Button appears for team leaders, modal shows summary

### Task 4: Add permission check for team leaders
**Type:** auto
**File:** routes.py
**Action:**
- Team leader = user.is_admin or user has group membership
- Add check in /filtered-summary route
**Verification:** Non-leaders get 403, leaders get summary

## Output Spec
- generate_filtered_summary function exists
- /filtered-summary route with permission check
- AI Summary button on manage_records.html
- Modal shows filter criteria and grouped summary

## Success Criteria
1. Team leader can click "AI Summary" button on filtered results
2. Summary modal shows filter criteria (users, groups, time range)
3. Multi-user summary groups content by user
