# Milestone v1.3 AI Requirements

## Overview

集成AI能力到周报管理系统，支持工作总结自动生成和周报文本润色功能。

**Core Value:** 让用户能够利用AI快速生成工作总结，提高周报编写效率。

---

## AI Configuration (CONFIG)

### CONFIG-01: AI服务配置存储

管理员可配置AI服务的API URL、API Key和模型名称，支持任意兼容OpenAI API格式的服务。

**Location:** 系统配置页面或新建专用AI配置页面

**Fields:**
- API URL (text) — 如 `https://api.openai.com/v1`
- API Key (text, masked) — 用户自行提供
- Model Name (text) — 如 `gpt-4o-mini`、`claude-3-5-haiku` 等

**Permissions:** 仅管理员可配置

**Validation:**
- API URL必须为有效URL格式
- API Key不能为空（保存时）
- 模型名称不能为空

---

### CONFIG-02: AI配置测试连接

管理员可测试配置的AI服务是否可用。

**Behavior:**
- 点击"测试连接"按钮
- 发送简单请求到配置的API（如获取模型列表或发送测试prompt）
- 显示成功/失败状态和错误信息

**Permissions:** 仅管理员可用

---

### CONFIG-03: AI配置持久化

AI配置保存到数据库或配置文件，系统重启后保持配置。

**Implementation:**
- 存储到config表或新建ai_config表
- API Key加密存储（不明文）

---

## Summary Templates (TEMPLATE)

### TEMPLATE-01: 模板存储和管理

管理员可创建、编辑、删除总结模板。

**Template Fields:**
- 模板名称 (text) — 如 "周总结模板"、"项目进展模板"
- 模板内容 (textarea) — 提示词模板，支持变量占位符
- 适用范围 (select) — 周/月/季度/年/通用

**Permissions:** 仅管理员可管理模板

**Database:** 新建 `ai_templates` 表

---

### TEMPLATE-02: 模板变量支持

模板支持占位符变量，系统在调用API时自动填充。

**Supported Variables:**
- `{time_range}` — 时间范围描述
- `{user_name}` — 用户姓名
- `{records}` — 周报内容列表
- `{record_count}` — 周报数量
- `{date_range}` — 具体日期范围

**Example Template:**
```
请为 {user_name} 生成一份 {time_range} 工作总结。
时间范围：{date_range}
共有 {record_count} 条周报记录：

{records}

请按以下格式输出：
1. 主要工作内容
2. 重点成果
3. 下一步计划
```

---

### TEMPLATE-03: 默认模板初始化

系统提供默认模板，管理员可修改或删除。

**Default Templates:**
- 周总结模板
- 月总结模板
- 季度总结模板
- 年总结模板

## Personal Summary (SUMMARY)

### SUMMARY-01: 主页一键生成工作总结

用户在主页可一键生成个人工作总结。

**Location:** 主页 (home.html)，在"最近提交"区域上方添加AI总结卡片

**UI Components:**
- 时间范围下拉选择：本周/本月/本季度/本年/自定义
- 模板下拉选择（可选）
- 自定义提示词输入框（可选）
- "生成总结"按钮
- 总结内容显示区域
- "复制"按钮
- "导出"按钮（可选导出为文本/PDF/DOCX）

**Data Source:** 基于当前登录用户在选定时间范围内的所有周报

**Permissions:** 所有用户可用（总结自己的工作）

---

### SUMMARY-02: 自定义提示词输入

用户生成总结时可输入自定义提示词，指导AI生成特定风格的总结。

**Behavior:**
- 提示词输入框为可选
- 如有输入，附加到模板或替代默认prompt
- 用户可保存常用提示词（可选功能）

---

### SUMMARY-03: 模板选择

用户可选择管理员配置的预设模板。

**Behavior:**
- 模板下拉显示所有可用模板
- 选择模板后显示模板名称和简要描述
- 默认选择与时间范围匹配的模板（如选择"本周"则默认"周总结模板")

---

### SUMMARY-04: 总结结果显示

生成的总结在页面显示，不自动保存到数据库。

**Display Format:**
- HTML富文本格式（支持段落、列表等）
- 或纯文本格式（用户可选）

**Actions:**
- 复制到剪贴板
- 导出为文件（文本/PDF/DOCX）

---

## Filtered Summary (FILTER-SUM)

### FILTER-SUM-01: 筛选结果AI总结

组长和管理员可在查找页对筛选结果进行AI总结。

**Location:** 查找页 (manage_records.html)，在筛选结果统计区域添加"AI总结"按钮

**Trigger:**
- 筛选后显示"AI总结筛选结果"按钮
- 点击后弹出模态框

**UI in Modal:**
- 显示筛选条件摘要（用户、组、时间范围）
- 模板选择（可选）
- 自定义提示词输入（可选）
- "生成总结"按钮
- 总结结果显示区域
- 复制/导出按钮

**Data Source:** 基于筛选出的周报（可能包含多个用户）

**Permissions:** 组长（view_group权限）和管理员（view_all权限）可用

---

### FILTER-SUM-02: 多用户总结格式

筛选结果包含多个用户时，总结格式清晰区分各用户工作。

**Format:**
- 按用户分组展示
- 或整体汇总（用户可选格式）

---

## Text Polish (POLISH)

### POLISH-01: 周报文本润色

用户在新建/编辑周报时可使用AI润色已输入的文本。

**Location:** 新建/编辑页 (create_records.html)，在CKEditor编辑器下方添加"润色"按钮

**Behavior:**
- 用户输入文本后点击"润色"按钮
- 发送当前编辑器内容到AI
- AI润色后替换编辑器内容
- 用户可继续编辑后再保存

**Default Prompt:** "请润色以下周报文本，保持原意，优化表达，使其更清晰专业："

**Permissions:** 所有用户可用

---

### POLISH-02: 润色提示词可配置

润色使用的提示词可在系统配置中修改。

**Configuration:**
- 管理员可修改默认润色提示词
- 用户可选择润色风格（可选功能）

---

## API Integration (API)

### API-01: OpenAI兼容API调用

系统使用配置的API URL/Key/Model调用AI服务，兼容OpenAI API格式。

**Implementation:**
- 使用 `openai` Python库或 `requests` 直接调用
- 调用 `/chat/completions` 端点
- 支持streaming响应（可选）

---

### API-02: API错误处理

系统妥善处理API调用失败情况。

**Error Cases:**
- 网络错误 — 显示"网络连接失败，请检查API URL"
- 认证错误 — 显示"API Key无效"
- 速率限制 — 显示"请求过于频繁，请稍后再试"
- 模型错误 — 显示"模型名称无效或不可用"
- 内容过长 — 显示"内容过长，请减少周报数量或缩短时间范围"

**User Feedback:**
- 所有错误显示友好的中文提示
- 不暴露技术细节给普通用户

---

### API-03: 超时处理

API调用设置合理超时时间，避免长时间等待。

**Timeout:**
- 默认30秒超时
- 用户可看到加载指示器
- 超时后显示友好提示

---

### API-04: 响应内容处理

系统处理AI返回的内容，确保格式正确。

**Processing:**
- 去除可能的多余空白
- 支持Markdown转HTML（如果AI返回Markdown）
- 确保输出为可显示的格式

---

## Security (SEC)

### SEC-01: API Key加密存储

API Key不在数据库明文存储。

**Implementation:**
- 使用加密存储（如Fernet对称加密）
- 密钥从环境变量获取
- 调用时解密使用

---

### SEC-02: API调用日志

系统记录AI调用日志用于审计和问题排查。

**Log Content:**
- 调用时间
- 用户
- 功能类型（总结/润色）
- 输入内容长度
- 响应状态（成功/失败）
- 错误信息（如有）
- 不记录完整prompt和响应内容（隐私）

---

### SEC-03: 权限控制

AI功能根据用户角色控制访问。

**Permission Matrix:**

| 功能 | 所有用户 | 组长 | 管理员 |
|------|---------|------|--------|
| AI配置 | ✗ | ✗ | ✓ |
| 模板管理 | ✗ | ✗ | ✓ |
| 个人总结 | ✓ | ✓ | ✓ |
| 筛选总结 | ✗ | ✓ | ✓ |
| 文本润色 | ✓ | ✓ | ✓ |

---

## UI/UX (UI)

### UI-01: 加载状态指示

AI调用期间显示加载指示器，用户知道系统正在工作。

**Implementation:**
- 显示spinner或进度条
- 禁用相关按钮防止重复点击
- 显示"正在生成..."提示

---

### UI-02: 结果交互

生成结果后提供便捷的后续操作。

**Actions:**
- 复制到剪贴板 — 一键复制
- 导出 — 可选导出格式
- 重新生成 — 使用相同参数重新生成
- 编辑后保存 — 用户可手动复制到新周报

---

---

## Out of Scope

以下功能不在v1.3范围内：

- **本地模型支持(Ollama)** — v1.3仅支持远程API，本地模型可在后续版本考虑
- **总结自动保存** — 总结仅显示不自动保存到数据库
- **总结历史记录** — 不记录用户的总结历史
- **批量润色** — 仅支持单条周报润色，不支持批量处理
- **多语言支持** — 提示词和输出均为中文
- **成本统计** — 不统计API调用成本
